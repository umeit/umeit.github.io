    Smartisan OS 使用体验

    本文写于 2016 年 4 月

    虽然在 2014 年就第一时间看了老罗的 Smartisan OS 的发布会，可真正日常使用是最近刚买了坚果以后。目前已上手使用了十几天，在体验 Smartisan OS 中哪些贴心的设计与养眼的 UI 的同时，也发现了一些设计上存在的缺陷和一些可以再优化的地方。

    说明
    优化：功能已完善，还可以继续完善
    改进：功能设计有问题，需要改进
    已修复：已在新版系统更新中修复

    通知中心

    隐藏「清空通知」按钮（改进 - 已修复）


    当通知中心列表页为空时，再点击「清空通知」按钮就变成了一个无效操作，应隐藏该按钮。

    通知中心内的 App 图标与用户选择的不一致（改进 - 已修复）


    当用户选择某个 App 的显示图标为重绘版的图标后，该 App 在通知中心的显示应该与用户选择的一致，否则将打破系统 UI 的一致性，并且对用户识别该应用也造成一定的干扰（有些重绘后的图标与原生图标差别较大）。

    App 通知声音支持自定义（优化）

    当用户装有大量 App 时，就会经常收到各类推送信息，对大多数用户来说虽然仅有少数几条信息是需要及时查看的，但在掏出手机点亮屏幕前无法做出判断。如果允许用户可以针对不同的 App 设置不同的提示音，那么当用户收到推送时就可以判断出是哪个 App 发来的推送，进而决定是否要及时查看。

    状态栏

    App 更新完成提示中的 icon 有歧义（改进）


    这里的「向下箭头」仅适用于表示「下载」的含义，而下载只是更新 App 过程中的一个步骤，所以用表示一个步骤的 Icon 来表示整个更新过程的完成是不够准确的。建议更改为「✔️」或其他更有代表性的 icon。

    拇指下拉悬停

    支持锁屏（优化）


    下拉悬停是一个帮助用户单手操作手机的实用功能，再进一步推进的话，用户在单手操作完手机后也有单手锁屏的需求，这时用户显然按不到位于屏幕右上角顶端的锁屏实体键，所以下拉悬停这个功能应不局限于屏幕内的像素，也可以把物理按键映射进来，使用户在整个操作手机的过程中完全使用单手。

    通讯录

    加入技术支持或售后的电话（改进）


    通讯录中默认的锤子科技官方技术支持或售后联系方式，一旦手机出现问题用户需要自行到网上搜索售后方式，这不仅给用户增加了麻烦还有可能导致用户搜到了非官方的售后/维修渠道导致其他问题。

    省电模式

    大段文字描述影响视觉效果（优化）


    「省电模式」下的系统主页 UI 非常简洁，但永远占据屏幕中央的大段文字说明除了略显突兀外，对已经使用过该功能用户来说价值也不大。建议仅在用户首次使用省电模式时完整显示，以后都折叠起来，用户可自行展开。

    应用商店

    聚光灯（优化）


    Smartisan OS 应用商店中的三个聚光灯是一个很有标志性的设计。如果进一步推敲拟物设计，放大趣味性，可以在打开应用商店时获取当天的日落时间（天气信息中有该数据），当用户在日落后第一次打开应用商店时，用半透明黑色遮罩整个页面，显示聚光灯开灯动画后将界面的遮罩移除，恢复正常的彩色界面。

    对已有重绘 icon 的 App，在详情页上加标识并可预览（优化）


    重绘 icon 是人们喜爱 Smartisan 的重要原因之一，所以这个优势应该在更多的地方强调。当这些用户在随意的浏览应用商店时，会有希望看到哪些 App 的 icon 已被重绘过的需求。从另一个方面看，一个 App 的 icon 被重绘过也说明了它是受欢迎的，这也可以促进用户下载试用。再进一步讲，有一些对审美非常专注的用户，他们在浏览应用商店时甚至有欣赏/发现重绘的漂亮 icon 的需求。

    缺失滚动条（改进）

    应用商店内的所有可滚动页面均没有滚动条。

    在「可更新」列表页就可以查看 App 的更新详情（优化）

    几乎每个App 的体积都在逐渐变大，用户已安装 App 的数量也在上升，这导致了用户更新 App 的成本也在变大，所以很有必要让用户在列表页就可以看到 App 的更新详情，从而帮助用户判断应该选择更新哪些 App。

    「应用信息」排版没有对齐（改进）

    设置

    「系统更新」icon 有歧义（改进）


    与状态栏的提示 icon 属于同类问题，「向下箭头」仅适用表示「下载」，用它来表示系统更新是不准确的。

    「侧键快捷方式设置」示意图有歧义（改进）


    红框里的内容在当前这个功能上有些画蛇添足，首先它与如何设置侧按钮没有直接关系，其次由于文字极小，用户完全看清这张图片的内容也要花费一些功夫。所以这种设计造成用户理解上增加难度的同时，还增加了不必要的操作成本。其实如果将红框中的图片删除，只保留手状示意图就完全可以解释该设置的意义。

    天气 App

    摄氏华氏切换（改进）


    绝大多数用户没有频繁切换摄氏和华氏单位的需求，甚至很多人一次也没有切换过，这样一个使用频次极低的功能却放在了非常醒目的位置，除了在视觉上不合理外，一旦用户发生误触（很有可能），就会导致用户的天气数据单位错误。

    刷新按钮（改进）


    虽然用户都希望看到当前的天气情况，但很少有人需要在已打开 App 后还不断的刷新。所以仅在每次启动 App 时获取最新的数据就满足需求了，刷新按钮的存在不仅没有意义，还会增加用户的使用负担，误以为每次都要手动的点一下该按钮。

    添加删除按钮（改进）


    添加删除城市按钮和刷新天气信息按钮是不同类型的操作，并不适相邻放置。做为对城市的操作，根据亲密性原则更适合放置在城市名的旁边，也就是导航栏的最右侧。

    阴影偏移量（优化）


    红框中的内容区域中包含了数字、符号、多行小号文本、日期等信息，这种情况下并不适合用偏移量很大的阴影效果来修饰，除了可能导致内容不易识别外还会让界面显得不那么整洁。

    浏览器

    手势切换页面（优化）

    Smartisan 的锁屏界面中有四指横扫屏幕切换壁纸的特性，而这种特性也适合在网页浏览的行为中，用户在用浏览网页时往往会有打开多个相关标签并来回切换的场景，这种特性可以提高这类中级用户的操作效率。

    Home 键

    Home 键的作用应该唯一（优化）


    Home 键做为手机上的最重要按键，应该始终保持唯一性，所以在「模块视图」中使用 Home 键，应该返回到启动「模块视图」前的桌面，或桌面的首屏。

    桌面图标

    音乐（优化）


    音乐 App 不仅是休闲类的 App 的代表，并且有着唱机的拟物化设计，为了增加趣味性，作为一个 OS 厂商可以顺水推舟地将 icon 动态的替换为当前播放音乐的专辑封面。
